01  ·  Problem

Detection is instant. Debugging is still slow.

But the investigation path changes every time depending on what the data shows. Teams must check multiple signals, assess scope, and explain the issue in plain language to non-technical stakeholders.

That last part does not scale with fixed rules.

02  ·  Solution

The key step is not running queries. It is deciding which ones to run.

Deterministic logic flags the suspicious country and source combination.

Nova receives the incident and a bounded query set. Based on what each query returns, Nova decides whether the next query is necessary or already answered.

Output: verdict, pattern, explanation, operational impact.

03  ·  Why Nova Pro V1?

Rules can say something is wrong. They cannot read the evidence and adjust.

Nova Pro V1 sits in the right position: stronger reasoning than Lite 2, materially cheaper than Pro 2. For bounded incident investigation, that is the correct trade-off.

It runs on Bedrock with no new infrastructure to justify. Deeper thinking levels are available when the evidence is genuinely ambiguous.

The reasoning happens through tool decisions, not extended compute.
